# SPEC.md — 시스템 물리 설계 사양서 (처음부터 끝까지)

> 모든 수치는 ① 팀 자체 실험(4세션), ② 2.4GHz 실내 전파 모델, ③ 레퍼런스(PMC Virtual Door 2025) 근거.
> "확정" = 이대로 설치. "권장" = 이 값으로 시작하되 현장 캘리브레이션으로 미세조정.

## 1. 설치 배치도

```
        복도 (폭 ~2m 가정)
  ───────────────────────────────────────
   바깥                            안(건물 내부)
        [수신기 A]───── 5m ─────[수신기 B]
         │← 문틀에서 30cm 이내      │← 막힌 벽에서 최소 1~2m 이격
         높이 1.3m (태그 높이와 동일)
  ───────────────────────────────────────
              ↑ 문(출입 경계)은 A 쪽에 위치
```

## 2. 하드웨어 설정 총괄표

| 항목 | 값 | 상태 | 근거 |
|------|----|------|------|
| 수신기 이격 거리 | **5m** | 확정 | 자체 실험(1/3/5m 비교): 1~3m는 피크 겹침, 5m에서 교차점 최명확 |
| 비콘 광고 주기 | **500ms** | 확정 | 자체 실험(100~1000ms): 500ms 이상 성능차 없음 → 배터리·비용 최적 |
| 비콘 TX Power | **-12dBm** | 권장 | 아래 링크버짓 참조. -4dBm은 감지 반경 과대 → 반사파 FP |
| 수신기 높이 | **1.2~1.4m** | 권장 | 태그(펜, 가슴 포켓)와 동일 높이 → 수직 경로손실 제거 |
| 문틀과의 거리 | **30cm 이내** | 권장 | 레퍼런스: 문 밀착일수록 RSSI 패턴 선명 (96.6% 달성 조건) |
| 벽과의 거리 | **≥1.5m** | 권장 | 자체 분석: 막힌 벽 반사파가 유령 피크 생성 |
| scan_window | **= scan_interval** | 확정(펌웨어 수정) | 듀티사이클 100% → 패킷 손실 35~40% → ~12% |
| 광고 채널 | **37/38/39 유지** | 권장 | 단일 채널은 페이딩 시 전멸 위험. 채널 편차는 SMA+RISE_GUARD로 흡수 |
| iBeacon UUID | 팀 고유 1개 | 확정 | 타사 비콘 필터링 1차 방어선 (2차: firefighters 등록제) |

## 3. TX Power 링크버짓 (-12dBm 근거)

실내 경로손실 모델 RSSI(d) ≈ TX − 40 − 10·n·log₁₀(d), n≈2.7(복도):

| 거리 | TX -4dBm | TX -12dBm |
|------|---------|-----------|
| 1m  | -44 | -52 |
| 3m  | -57 | -65 |
| 5m  | -63 | -71 |
| 8m  | -68 | -76 |
| 12m | -73 | **-81 ← FLOOR(-80) 밖** |

- -12dBm이면 감지 반경이 **약 8~10m**로 줄어 두 수신기의 "동시 강수신 구간"이 좁아지고 원거리 반사파가 FLOOR 아래로 가라앉음
- -4dBm은 반경 20m+ → 옆 복도·반사파까지 다 잡힘 (FP의 주요 원인)
- ⚠️ 몸에 가리면(섀도잉) 10~20dB 추가 감쇠 → -20dBm까지 내리면 Miss 위험. **-12dBm에서 시작, Miss 발생 시에만 -8dBm로 한 단계 상향**

## 4. 소프트웨어 파라미터 (pi/params.py와 동일해야 함)

| 파라미터 | 값 | 역할 |
|---------|----|------|
| RSSI_FLOOR | -80 (현장 캘리브레이션) | 노이즈 컷 |
| SMA_WINDOW | 3 | 500ms 기준 이동평균 |
| PROMINENCE | 1.0 | 피크 최소 돌출 |
| MIN/MAX_CROSS_SEC | 2.0 / 9.0 | 5m·보행속도 0.5~2.5m/s 대응 |
| COOLDOWN_SEC | 5.0 | 꼬리 피크 차단 |
| RISE_GUARD_DB | 6.0 | 정지 핑퐁 FP 차단 (신규, 시뮬 검증) |
| EPISODE_QUIET_SEC | 2.5 | 에피소드 종료 → 판정 지연 = 통과 후 ~2.5초 |

## 5. 현장 캘리브레이션 절차 (설치할 때마다, 10분)
1. 수신기 설치 후 **태그 없이 5분** 수집 → 배경 RSSI 최대값 확인 → `RSSI_FLOOR = 배경최대 + 5dB` (기본 -80에서 조정)
2. 태그 들고 교차점에 서서 A/B 값이 -65±5 부근인지 확인 (아니면 TX/높이 점검)
3. 바깥→안 1회 통과 → `ENTRY` 확인. 반대면 `.env` A/B 교체 또는 `SWAP_AB` 토글
4. 검수 통과 기준: 보행 5회+구보 5회 → 감지 10/10, 방향 10/10, FP 0
